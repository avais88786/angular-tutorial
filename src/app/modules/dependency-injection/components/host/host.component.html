using @Host Resolution Modifier

Using @Host modifier asks angular to look into this components Metadata info as last option while trying to resolve dependency. Even if there are more parents to traverse or other methods angular wont search any further and return any defined providers in component metadata or error if nothing defined in this components metadata. If there were children from this components and their dependency resolution would bring angular to this component, angular wont go any further. Can be combined with @Optional()

Original value of emoji coming from leaf service implementation: {{mainLeaf.emoji}}
Overriden value of emoji, overriden in this components metadata info: {{leaf.emoji}}
