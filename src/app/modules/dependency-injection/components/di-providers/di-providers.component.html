DI Providers

Getting value using Aliased class provider:

Two providers exist, NewLoggerService and OldLoggerService, whevever an instance of OldLoggerService is requested, a NewLoggerService instance is returned instead.
Value of getLogMessage from oldMessageLogger is : {{loggerService.getLogMessage()}}

Although angular does not support dependency injecting of an interface (javascript does not know what an interface is) If defined in TypeScript an interface 'ILogger', we can define provider in module such that when something requests an implementation instance of this interface, a concrete implementation is injected.
Value from iLogger : {{iLogger.getLogMessageFromInterface()}}

Getting value using useValue provider:

use ‘useValue’ if your stub is a object, ‘useClass’ if your stub is a class. The only difference is ‘useClass’ does the whole instantiation process necessary for using a class. https://angular.io/guide/dependency-injection-in-action#useclassthe-class-provider
Here, an interface is injected with useValue Provider, the value of useValue provider is an object, one can use useClass here but then angular instantiates the class Value from ILoggerWithUseValue: {{iLoggerWithValue.log()}}

Using Service Provider factories:

Value coming from an interface which is implemented by multiple concrete implementations. NgModule uses factory to provide an instance relevant at runtime.
Value from serviceWithFactoryprovider: {{serviceWithFactoryprovider1.getValue()}}

Using 'multi:true in provider token'

By using
{ provide: 'MultiInstances', useExisting: NewLoggerService, multi: true }
{ provide: 'MultiInstances', useExisting: OldLoggerService, multi: true }
we are saying there are multiple instances to be injected if someone requests injector with a token 'MultiInstances'
